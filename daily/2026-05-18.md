# Daily Research Radar — 2026-05-18

## Highlights

- **CFD surrogate 쪽은 “conditioning을 모르는 모델”에서 벗어나는 신호가 강하다.** self-supervised fluid inpainting, shock-centered neural-operator representation, mesh spatio-temporal awareness는 geometry/BC/registration/temporal context를 explicit하게 다루려는 흐름이다.
- **Turbulence closure는 물리 제약 없는 symbolic/ML discovery만으로는 위험하다는 메시지가 선명하다.** realizability-constrained wake-flow closure는 learned closure를 solver 안에서 안정적으로 쓰려면 non-physical behavior를 제약해야 함을 보여준다.
- **Thermal/energy control은 active-learning digital twin으로 좁혀 읽는 게 좋다.** integrated thermal energy system 논문은 Modelica simulation, surrogate, uncertainty, active learning을 supervisory control 목적에 묶는다.
- **CAD automation은 benchmark보다 “tool surface”가 빠르게 늘고 있다.** KiCAD, FreeCAD, multiCAD, Fusion 360, Rhino MCP가 동시에 나오고 있어 CAD agent의 다음 병목은 생성 능력보다 artifact validation/contract가 될 가능성이 높다.
- **저신호 범용 AI 논문은 제외했다.** 오늘은 sciML/CFD/CAD/control에 직접 닿는 논문·도구 위주로만 추렸다.

## Added Resources

### CFD surrogates, neural operators & turbulence closure

- [Inpainting physics: self-supervised learning for context-driven fluid simulation](https://arxiv.org/abs/2605.08832) — CFD surrogate를 geometry/BC 명시 입력에만 묶지 않고 주변 context에서 missing flow field를 복원하는 방식으로 학습한다. arbitrary inlets/obstacles/sensor gaps 같은 실제 설계 상황에 더 가까운 conditioning 연구로 읽을 만하다. `#CFD` `#self-supervised` `#surrogate` `#conditioning`
- [Shock-Centered Low-Rank Structure and Neural-Operator Representation of Rarefied Micro-Nozzle Flows](https://arxiv.org/abs/2605.12723) — DSMC rarefied micro-nozzle flow에서 shock를 중심으로 정렬하면 apparent complexity가 크게 줄어든다는 점을 neural-operator representation과 연결한다. compressible/rarefied flow surrogate에서 registration이 모델 크기만큼 중요하다는 신호다. `#neural-operator` `#shock` `#rarefied-flow` `#low-rank`
- [Realizability-Constrained Machine Learning for Turbulence Closures in Wake Flows](https://arxiv.org/abs/2605.12304) — symbolic-regression/ML turbulence closure가 solver에서 non-physical behavior와 residual stagnation을 만들 수 있다는 문제를 realizability constraint로 다룬다. learned closure를 실제 CFD loop에 넣으려면 stability/realizability gate가 필수라는 좋은 근거. `#turbulence` `#closure-model` `#CFD` `#physics-constraints`
- [Mesh Based Simulations with Spatial and Temporal awareness](https://arxiv.org/abs/2605.01542) — mesh-based CFD surrogate에서 architecture만 비교하지 말고 spatial/temporal awareness를 명시적으로 넣어야 한다는 문제 제기. GNN/Transformer surrogate의 rollout 안정성과 mesh 일반화 평가를 설계할 때 참고할 만하다. `#mesh` `#GNN` `#CFD-surrogate` `#rollout`

### Thermal systems, control & digital twins

- [Physics-based Digital Twins for Integrated Thermal Energy Systems Using Active Learning](https://arxiv.org/abs/2605.06756) — Modelica thermal-system simulation에 SINDyC/MvG-SINDyC/FNN/GRU surrogate와 active learning을 결합해 glycol heat-exchanger supervisory control을 겨냥한다. heat-transfer/control 관심사에 직접 맞는 응용 anchor. `#thermal` `#digital-twin` `#active-learning` `#control`
- [Real-time control of multiphase processes with learned operators](https://arxiv.org/abs/2603.25308) — multiphase process의 strong nonlinearity, phase transition, sparse sensing 문제를 learned operator 기반 실시간 제어로 다룬다. surrogate를 prediction-only가 아니라 closed-loop/MPC 안에 넣는 방향의 좋은 reference. `#control` `#multiphase-flow` `#learned-operator` `#MPC`

### CAD, MCP & agentic engineering tools

- [KiCAD-MCP-Server](https://github.com/mixelpixx/KiCAD-MCP-Server) — LLM이 KiCAD PCB 프로젝트를 직접 inspect/edit할 수 있게 하는 MCP server. CAD/EDA agent tooling이 “대화형 조언”에서 실제 design artifact 조작으로 내려오는 신호다. `#MCP` `#KiCAD` `#EDA` `#agentic-engineering`
- [neka-nat/freecad-mcp](https://github.com/neka-nat/freecad-mcp) — FreeCAD MCP server. parametric CAD를 agent-callable operation surface로 노출하는 대표적인 오픈소스 후보라, CAD→mesh→CFD 자동화 실험의 tool 후보로 추적 가치가 높다. `#FreeCAD` `#MCP` `#CAD-automation`
- [multiCAD-mcp](https://github.com/AnCode666/multiCAD-mcp) — AI assistant가 여러 CAD software를 다루도록 하는 MCP server. maturity는 더 봐야 하지만 vendor/tool별 MCP bridge가 난립하기 시작했다는 생태계 신호로 중요하다. `#MCP` `#CAD` `#workflow`
- [FreeCAD AI workbench](https://github.com/ghbalf/freecad-ai) — 자연어로 FreeCAD 모델을 생성하는 AI assistant workbench. 생성 품질 검증은 별도지만, FreeCAD 내부 workbench 형태라 CAD automation UX 참고자료로 좋다. `#FreeCAD` `#text-to-CAD` `#design-automation`

## Best Find of the Day

**Inpainting physics**. 오늘의 핵심은 모델이 정해진 geometry/BC vector만 받는 forward surrogate에서 벗어나, 실제 엔지니어링 상황처럼 일부 context와 sparse observation으로 field를 재구성하는 방향이다. CAD/CFD workflow에서는 항상 완벽한 입력 schema가 있는 게 아니라 geometry patch, 센서, 부분 해석 결과, 이전 iteration artifact가 섞이므로 이 framing이 꽤 실용적이다.

## Possible Research Ideas

1. **CFD inpainting mini-benchmark:** lid-driven cavity/duct flow에서 obstacle mask, inlet profile 일부, sensor probes만 주고 missing field를 복원하는 작은 benchmark를 만든다. L2뿐 아니라 boundary-layer/shock/interface error를 따로 본다.
2. **Shock registration before neural operators:** shock tube나 nozzle surrogate에서 raw coordinate 학습 vs shock-centered coordinate/feature 학습을 비교해 “전처리/registration이 architecture보다 이득인지” 확인한다.
3. **CAD MCP validation contract:** FreeCAD MCP로 간단한 duct/heat-sink를 만들고, named faces, bounding box, STEP export, meshability, solver stub log까지 필수 evidence로 남기는 agent contract를 정의한다.
