# Daily Research Radar — 2026-05-11

## Highlights

- **CFD agent workflow가 “아이디어 생성”에서 solver-grounded discovery loop로 넘어가고 있다.** AI CFD Scientist는 문헌 조사, OpenFOAM 실행, mesh-independence check, physics/vision 진단, 코드 수정, 논문 초안까지 artifact contract로 묶는다.
- **CFD surrogate는 대형 3D aero와 solver-in-the-loop 가속으로 양분된다.** AeroJEPA/foundation-style aero는 latent representation scale-up을, FNO-LBM/radiation-transfer NO는 기존 solver 내부 병목을 치환하는 방향을 보여준다.
- **Thermal/heat-transfer SciML은 건물·배터리·plasma·fire처럼 실제 제어/안전/설계 의사결정과 붙는 사례가 계속 늘고 있다.** 단순 heat equation benchmark보다 system-specific constraints와 property/radiation bottleneck을 보는 편이 더 유용하다.
- **CAD/CAE agent 쪽의 핵심은 API surface와 provenance다.** FreeCAD/KOMPAS/SolidWorks 계열 MCP·COM 자동화 신호는 “말로 CAD 생성”보다 deterministic operation, drawing/export, BC tagging, validation hook을 노출하는 쪽이 중요하다는 점을 강화한다.
- **산업 도입 관점에서는 verification, auditability, human-in-the-loop가 모델 성능보다 더 큰 blocker다.** Agentic AI in Engineering and Manufacturing 인터뷰 논문은 VA의 agent workflow 설계 원칙을 뒷받침하는 좋은 근거다.

## Added Resources

### CFD agents & OpenFOAM workflows

- [AI CFD Scientist](https://arxiv.org/abs/2605.06607) · [code](https://github.com/csml-rpi/AI-CFD-Scientist) — 문헌 기반 아이디어 생성부터 OpenFOAM 실행, mesh-independence check, diagnostics, cross-case analysis, LaTeX draft까지 연결하는 open-source CFD scientist workflow. `#OpenFOAM` `#agents` `#CFD`
- [Agentic AI in Engineering and Manufacturing](https://arxiv.org/abs/2604.09633) — 30+ industry stakeholder interview 기반으로 engineering/manufacturing agent 도입의 가치와 병목을 정리. verification, auditability, fragmented legacy data/API 문제가 핵심이다. `#engineering-agents` `#workflow` `#adoption`

### SciML, neural operators & CFD surrogates

- [Hybrid Fourier Neural Operator-Lattice Boltzmann Method](https://arxiv.org/abs/2604.27158) — FNO를 LBM 초기화/rollout coupling에 넣어 steady/unsteady weakly-compressible flow를 가속. solver 대체가 아니라 solver 내부 가속 패턴이라 실용성이 높다. `#FNO` `#LBM` `#solver-acceleration`
- [Nested Fourier-enhanced neural operator for radiation transfer in fires](https://arxiv.org/abs/2604.13919) — fire CFD에서 radiative transfer equation 병목을 Fourier-MIONet 계열 neural operator로 치환하려는 논문. thermal/fire simulation surrogate로 강한 신호. `#heat-transfer` `#radiation` `#fire-CFD`
- [Reduced-Order Modeling of Parameterized Visco-Plastic Shallow Flows](https://arxiv.org/abs/2605.06526) — Herschel-Bulkley shallow flow의 moving front/yield surface를 tensor ROM으로 빠르게 재구성. neural operator가 아니어도 구조화된 parameter space에서는 tensor ROM이 여전히 유효하다. `#ROM` `#non-Newtonian` `#surrogate`
- [Toward Generalizable Graph Learning for 3D Engineering AI](https://arxiv.org/abs/2604.07781) — CAD/FE/CFD mesh 등 heterogeneous 3D engineering asset을 physics-aware graph로 바꿔 CAE mode classification과 CFD field prediction에 적용. `#GNN` `#CAE` `#3D-engineering-AI`
- [DeepPropNet](https://arxiv.org/abs/2604.27298) — thermal plasma의 thermodynamic/transport property를 operator learning으로 예측. 반복적인 property evaluation이 병목인 multiphysics simulation에 붙이기 좋다. `#thermal-plasma` `#operator-learning` `#properties`

### CAD/CAE automation tools

- [KOMPAS-3D MCP client](https://github.com/dwnmf/KOMPAS-3D-MCP-bin) — KOMPAS-3D를 MCP client로 연결하는 Windows 패키지. VA 환경의 직접 의존성은 아니지만 CAD ecosystem이 MCP surface를 노출하는 흐름을 보여준다. `#CAD` `#MCP` `#agent-tools`
- [SolidWorks CAD Automation Pipeline](https://github.com/wikijerry/CAD-Automation-pipeline) — SolidWorks 3D → 2D drawing generation, auto-dimensioning, view selection, validation을 Python COM API와 LLM으로 묶는 초기 prototype. 아직 낮은 성숙도지만 drawing/documentation automation 신호로 볼 만하다. `#SolidWorks` `#CAD-automation` `#LLM-agents`

## Best Find of the Day

**AI CFD Scientist**가 오늘의 best pick. 새 모델 하나보다 더 중요한 것은 CFD agent가 어떤 artifacts를 남기고, 어디서 physics validity를 확인하며, 실패했을 때 어떻게 진단/수정하는가다. 이 프로젝트는 `lit.json`/`requirements.json`/mesh spec/analysis artifact 같은 계약을 전면에 놓기 때문에, VA의 OpenFOAM agent harness와 sim-benchmark식 provenance 평가를 설계할 때 직접 참고할 만하다.

## Possible Research Ideas

1. **OpenFOAM agent artifact schema 비교:** AI CFD Scientist, sim-benchmark, FoamPilot의 output contract를 비교해서 VA용 최소 schema(`case_manifest.json`, `mesh_check.json`, `kpi_provenance.json`, `physics_checks.json`)를 정의한다.
2. **Radiation/property surrogate sandbox:** full CFD solver를 대체하지 말고 radiation transfer 또는 material/property evaluator만 neural operator로 치환하는 작은 heat-transfer benchmark를 만든다.
3. **CAD-to-drawing-to-CAE validation loop:** FreeCAD/SolidWorks agent가 geometry뿐 아니라 drawing, named faces, BC tags, STEP/STL export, meshability report까지 남기도록 checklist를 설계한다.
