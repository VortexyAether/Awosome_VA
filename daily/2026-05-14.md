# Daily Research Radar — 2026-05-14

## Highlights

- **Neural-operator 쪽은 “큰 universal surrogate”보다 compositionality, geometry charting, JAX/PDE residual plumbing처럼 실제 연구 루프를 버티는 구조가 중요해지고 있다.** Compositional NO, CATO, jNO가 각각 multi-dimensional fluids, complex geometry attention, reproducible training stack을 건드린다.
- **Thermal/control 응용은 active learning과 interpretable/probabilistic surrogate가 강한 신호다.** Integrated thermal-energy digital twin 논문은 random sampling 대비 훨씬 적은 simulation trajectory로 heat-exchanger control surrogate를 만든다.
- **CAD automation 평가는 예쁜 shape generation에서 executable tests와 operation fidelity로 이동 중이다.** CADTests와 BenchCAD 계열은 CadQuery program, deterministic tests, edit fidelity를 전면에 둔다.
- **Agentic engineering workflow의 다음 bottleneck은 tool 선택·권한·검증이다.** ToolCUA/MCP authorization 논문은 CAD/CAE agent가 GUI와 API/MCP tool 사이를 안전하게 오가야 한다는 현실적인 문제를 짚는다.
- **CFD surrogate는 여전히 validation red-team이 필요하다.** 최근 inpainting/latent/operator papers가 좋아도, unstable-flow hallucination류의 failure mode를 digest/benchmark에 같이 붙여야 과장된 “AI solver” 프레이밍을 피할 수 있다.

## Added Resources

### Neural operators, SciML infrastructure & CFD surrogates

- [Compositional Neural Operators for Multi-Dimensional Fluid Dynamics](https://arxiv.org/abs/2605.11691) — multi-dimensional fluid dynamics를 대상으로 operator components를 조합하는 방향. “하나의 거대한 universal model”보다 task/geometry/field structure를 조합 가능한 module로 다루려는 신호라, CFD surrogate generalization 논의에 좋다. `#neural-operator` `#CFD` `#fluid-dynamics` `#compositionality`
- [jNO: A JAX Library for Neural Operator and Foundation Model Training](https://arxiv.org/abs/2605.10159) · [code](https://github.com/FhG-IISB/jNO) — domain/model/residual/loss tracing을 갖춘 JAX-native neural-operator training library. paper-only method보다 VA 실험 루프에 붙이기 쉬운 infrastructure signal이다. `#JAX` `#neural-operator` `#SciML-tooling`
- [CATO: Charted Attention for Neural PDE Operators](https://arxiv.org/abs/2605.09016) — complex geometry에서 Transformer-style PDE operator가 겪는 coordinate/attention 문제를 charted attention으로 다루려는 시도. CAD/mesh geometry가 들어가는 surrogate에서 눈여겨볼 만하다. `#PDE-operator` `#complex-geometry` `#attention`
- [Physics-Informed Neural PDE Solvers via Spatio-Temporal MeanFlow](https://arxiv.org/abs/2605.08915) — pointwise residual 중심 PINN/NO 한계를 보완하려는 spatio-temporal MeanFlow 기반 PDE solver. long-range dependency와 continuous physics를 다루는 쪽의 method candidate. `#PINN` `#MeanFlow` `#PDE-solver`
- [AeroJEPA: Learning Semantic Latent Representations for Scalable 3D Aerodynamic Field Modeling](https://arxiv.org/abs/2605.05586) — 3D aerodynamic field surrogate에서 latent representation을 field reconstruction뿐 아니라 interpolation/probing/design optimization에 쓰려는 접근. 설계 변수 탐색용 latent space 관점에서 계속 추적 가치가 있다. `#aerodynamics` `#3D-surrogate` `#latent-space`

### Thermal/control digital twins

- [Physics-based Digital Twins for Integrated Thermal Energy Systems Using Active Learning](https://arxiv.org/abs/2605.06756) — Modelica 기반 thermal system simulation에 SINDyC/MvG-SINDyC/FNN/GRU surrogate와 active learning을 붙여 glycol heat-exchanger supervisory control을 겨냥한다. thermal/heat-transfer 쪽에서는 오늘 가장 직접적인 응용 anchor. `#thermal` `#digital-twin` `#active-learning` `#control`
- [Fast and Accurate Prediction of Lattice Thermal Conductivity via Machine Learning Surrogates](https://arxiv.org/abs/2605.11610) — lattice thermal conductivity prediction을 ML surrogate로 가속하려는 materials/thermal paper. CFD는 아니지만 heat-transfer material screening과 thermal design loop에 붙을 수 있다. `#thermal-conductivity` `#materials` `#surrogate`

### CAD automation, MCP & agentic engineering workflows

- [Text-to-CAD Evaluation with CADTests](https://arxiv.org/abs/2605.07807) · [code](https://github.com/dimitrismallis/CADTestBench) — generated CAD를 executable CadQuery tests로 평가. VA의 CAD→mesh→CFD workflow에도 “prompt/spec → CAD artifact → deterministic test → repair” 패턴을 그대로 가져올 수 있다. `#CAD` `#CadQuery` `#evaluation` `#design-automation`
- [ToolCUA: Towards Optimal GUI-Tool Path Orchestration for Computer Use Agents](https://arxiv.org/abs/2605.12481) — computer-use agent가 GUI action과 high-level tool/API call 사이를 언제 전환해야 하는지 다룬다. FreeCAD/SolidWorks GUI와 MCP/API를 섞는 engineering agent 설계에 간접적으로 유용하다. `#agentic-workflow` `#GUI-agent` `#tool-use`
- [Options, Not Clicks: Lattice Refinement for Consent-Driven MCP Authorization](https://arxiv.org/abs/2605.11360) — MCP tool invocation에서 broad allow 대신 argument-aware consent lattice를 제안. CAD/CAE agent가 파일 export, solver run, cloud upload 같은 위험도 다른 operation을 안전하게 다루는 데 참고할 만하다. `#MCP` `#authorization` `#agent-safety`

## Best Find of the Day

**jNO + Compositional Neural Operators 조합**. 하나는 실제 JAX 기반 neural-operator 실험 infrastructure, 하나는 multi-dimensional fluid surrogate의 compositional modeling 방향이다. VA 관점에서는 “논문 하나 읽기”보다, 작은 CFD/thermal benchmark를 만들고 여러 operator architecture를 빠르게 바꿔 끼우는 실험 루프를 만드는 쪽이 더 가치 있다.

## Possible Research Ideas
