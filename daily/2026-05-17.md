# Daily Research Radar — 2026-05-17

## Highlights

- **Neural operator 쪽 신호는 “표현 공간을 어떻게 나눌 것인가”로 이동 중이다.** Hodge decomposition, cross-domain operator, frequency-bias 분석은 단순 FNO scaling보다 topology/geometry/spectrum을 분리해 다루는 쪽이 중요하다는 근거다.
- **CFD surrogate는 interface와 conditioning 문제가 계속 핵심이다.** ViT-K는 Stokes/Navier–Stokes–Darcy coupled flow의 interface condition을 few-shot으로 다루고, 이는 porous/thermal-fluid 설계 surrogate와 연결된다.
- **Control은 learned operator를 prediction-only 모델이 아니라 real-time decision loop에 넣는 방향이 강하다.** multiphase process control 논문은 sensor 제약과 nonlinear phase dynamics를 operator surrogate + control loop 관점에서 읽을 만하다.
- **Engineering agents는 CAD API/MCP와 simulation verification이 함께 가야 한다.** Jarvis Onshape MCP와 FreeCAD+FluidX3D/OpenFOAM MCP는 “agent가 CAD를 만진다”를 실제 CAD/CFD tool surface로 내리는 사례다.
- **Agentic SciML은 materials 쪽에서 먼저 workflow 형태가 선명해지고 있다.** Lang2MLIP는 active-learning MLIP 개발을 language-to-workflow agent로 묶어, CFD/SciML campaign automation에도 참고할 만한 패턴을 준다.

## Added Resources

### Neural operators, CFD surrogates & SciML foundations

- [Topology-Preserving Neural Operator Learning via Hodge Decomposition](https://arxiv.org/abs/2605.13834) — geometric mesh field equation에서 Hodge orthogonality로 topological degrees of freedom과 learnable geometric dynamics를 분리하려는 operator-level decomposition. unstructured CFD/CAE mesh surrogate에서 “보존해야 할 구조”를 모델에 어떻게 넣을지 힌트가 있다. `#neural-operator` `#geometry` `#topology` `#mesh`
- [UFO: A Domain-Unification-Free Operator Framework for Generalized Operator Learning](https://arxiv.org/abs/2605.12700) — physical/spectral/latent representation을 하나로 억지 통일하지 않고 adaptive cross-domain interaction으로 operator를 구성한다. FNO류의 single-domain bias를 줄이는 방향이라 multi-physics CFD/thermal surrogate에 참고 가능. `#operator-learning` `#cross-domain` `#SciML`
- [Frequency Bias and OOD Generalization in Neural Operators under a Variable-Coefficient Wave Equation](https://arxiv.org/abs/2605.12997) — neural operator가 structured distribution shift에서 어떤 frequency component를 선호/실패하는지 분석한다. CFD surrogate benchmark에서도 “평균 L2 error” 말고 spectral/OOD diagnostics가 필요하다는 좋은 근거. `#neural-operator` `#OOD` `#spectral-bias` `#validation`
- [ViT-K: A Few-Shot Learning Model for Coupled Fluid-Porous Media Flows with Interface Conditions](https://arxiv.org/abs/2605.13912) — Stokes/Navier–Stokes–Darcy coupled flow와 interface heterogeneity를 few-shot surrogate로 다룬다. cold plate, filtration, porous heat exchanger처럼 free-flow/porous interface가 있는 thermal-fluid 설계에 직접적이다. `#CFD` `#porous-media` `#few-shot` `#interface`

### Control, thermal-fluid systems & simulation workflows

- [Real-time control of multiphase processes with learned operators](https://arxiv.org/abs/2603.25308) — 강한 비선형성, phase transition, sensor 해상도 제약이 있는 multiphase process를 learned operator로 실시간 제어하려는 연구. surrogate를 단순 예측기가 아니라 MPC/closed-loop controller 안에 넣는 방향으로 읽어야 한다. `#control` `#multiphase-flow` `#learned-operator` `#MPC`
- [Chrono-Gymnasium: An Open-Source, Gymnasium-Compatible Distributed Simulation Framework](https://arxiv.org/abs/2605.14911) — Project Chrono 기반 high-fidelity multibody dynamics를 Ray로 분산 실행하고 Gymnasium interface로 감싼 framework. CFD 자체는 아니지만, solver-backed RL/optimization benchmark를 engineering agent workflow로 묶는 패턴이 좋다. `#simulation` `#RL` `#mechanical-systems` `#workflow`
- [Lang2MLIP: End-to-End Language-to-Machine Learning Interatomic Potential Development with Autonomous Agentic Workflows](https://arxiv.org/abs/2605.14527) — atomistic simulation/active learning/ML workflow를 language-driven agent가 조율하는 materials SciML 사례. VA 관점에서는 “agent가 실험 campaign을 설계·실행·검증한다”는 workflow pattern을 CFD surrogate campaign에 이식할 수 있다. `#agentic-SciML` `#materials` `#active-learning` `#workflow`

### CAD, MCP & agentic engineering tools

- [Jarvis Onshape MCP](https://github.com/ReshefElisha/jarvis-onshape-mcp) — Onshape CAD를 Claude Code/MCP에서 조작하는 plugin. vision decomposition, FeatureScript, parametric iteration을 묶어 “실제 CAD SaaS를 agent-callable surface로 노출”하는 좋은 사례다. `#CAD` `#Onshape` `#MCP` `#agentic-design`
- [sandraschi/freecad-mcp](https://github.com/sandraschi/freecad-mcp) — FreeCAD MCP server + webapp에 FluidX3D/OpenFOAM CFD extension을 붙이려는 early project. 낮은 maturity지만 CAD→CFD 자동화 interface가 어디로 가는지 보여주는 신호다. `#FreeCAD` `#MCP` `#OpenFOAM` `#FluidX3D`

## Best Find of the Day

**Topology-Preserving Neural Operator Learning via Hodge Decomposition**. 오늘 CAD/MCP tooling도 흥미롭지만, SciML 연구 관점에서는 이 논문이 더 오래 남을 수 있다. CAD/CAE/CFD surrogate가 unstructured mesh와 complex geometry로 갈수록, 단순히 더 큰 operator를 쓰는 것보다 divergence/curl/topological mode처럼 보존해야 할 구조를 분리하는 모델링이 중요해진다.

## Possible Research Ideas

1. **Spectral/OOD validation pack:** 2D heat equation, variable-coefficient wave, lid-driven cavity surrogate에 대해 L2뿐 아니라 frequency-band error, boundary-layer error, OOD coefficient shift를 같이 기록하는 작은 benchmark를 만든다.
2. **Free-flow/porous interface mini benchmark:** Stokes–Darcy toy case나 cold-plate porous insert geometry로 interface-condition-aware surrogate baseline을 만든다.
3. **CAD-to-CFD MCP smoke test:** Onshape/FreeCAD MCP로 간단한 duct/heat-sink geometry를 생성하고, STEP export → meshability check → OpenFOAM/FluidX3D stub run까지 evidence log를 남기는 최소 pipeline을 만든다.
