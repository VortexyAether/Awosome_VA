# Daily Research Radar — 2026-05-10

## Highlights

- **CFD surrogate 쪽은 “자동차/난류/센서-to-field”처럼 실제 engineering loop에 가까운 benchmark로 이동 중이다.** RETO, LESnets, rt-EBIV가 모두 단순 PDE toy보다 고해상도 field, wall turbulence, real-time measurement reconstruction을 겨냥한다.
- **Thermal SciML은 chip/building처럼 제어·설계 의사결정에 바로 붙는 방향이 강하다.** 2D-ThermAl과 foundational building thermal model은 transient heat map과 control-aware thermal dynamics를 빠르게 예측하려는 흐름이다.
- **ROM/control 논문들은 “새 조건에서 다시 시뮬레이션 없이 적응”을 문제로 삼는다.** cXGBoost ROM adaptation과 CNMc는 optimization/control loop에 넣기 쉬운 lightweight surrogate layer로 볼 수 있다.
- **Engineering agents는 CAD/CAE를 말로 조작하는 데서 끝나지 않고, solver artifact와 provenance를 검증하는 쪽으로 가야 한다.** sim-benchmark의 source-provenance scoring은 OpenFOAM agent 평가 기준으로 꽤 좋은 신호다.
- **FreeCAD MCP 계열은 빠르게 분화 중이다.** Tessa Labs freecad-mcp는 CAD 생성뿐 아니라 defeaturing, BC tagging, CAE handoff까지 표면화하려는 점이 VA의 CAD→mesh→simulation automation 관심사와 잘 맞는다.

## Added Resources

### SciML, neural operators & CFD surrogates

- [RETO: A Rotary-Enhanced Transformer Operator for High-Fidelity Prediction of Automotive Aerodynamics](https://arxiv.org/abs/2605.00062) — DrivAerML/ShapeNet에서 automotive aerodynamic field를 예측하는 Transformer operator. RoPE 기반 spatial relation encoding을 neural operator에 넣어 local gradient와 relative displacement를 더 잘 잡으려는 방향이다. `#CFD` `#neural-operator` `#aerodynamics`
- [LESnets: Large-eddy simulation nets based on physics-informed neural operator](https://arxiv.org/abs/2604.26621) — 3D wall-bounded turbulence에서 PINO의 long-term stability와 multi-scale vortex 예측을 LES-style 구조로 보강하려는 논문. wall turbulence surrogate 평가 후보로 적합하다. `#turbulence` `#PINO` `#LES`
- [Real-Time Estimation of High-Resolution Flow Fields and Reduced-Order Coordinates from Event-Based Imaging Velocimetry](https://arxiv.org/abs/2605.04186) — event-based velocimetry의 coarse/low-latency 관측을 HR field와 POD 좌표로 복원. sensor-to-ROM, closed-loop flow control pipeline에 연결하기 좋다. `#ROM` `#sensing` `#flow-control`
- [Hybrid Fourier Neural Operator-Lattice Boltzmann Method](https://arxiv.org/abs/2604.27158) — FNO를 LBM의 초기화/rollout coupling에 넣어 solver를 완전히 대체하지 않고 가속하는 접근. “ML inside trusted solver” 패턴이라 실험 설계에 참고할 만하다. `#FNO` `#LBM` `#solver-acceleration`

### Thermal, ROM & control-oriented modeling

- [2D-ThermAl: Physics-Informed Framework for Thermal Analysis of Circuits using Generative AI](https://arxiv.org/abs/2512.01163) — power map/physical inputs에서 chip transient/steady thermal distribution과 heat-source localization을 추정. electronics thermal design의 early-stage screening에 유용한 방향이다. `#thermal` `#heat-transfer` `#electronics`
- [Constrained Extreme Gradient Boosting for Adapting Reduced-Order Models](https://arxiv.org/abs/2605.04130) — parameter variation에서 ROM basis 성능이 무너지는 문제를 constrained XGBoost로 보정. CFD/FEA optimization loop에서 deep model보다 가볍게 붙일 수 있는 adaptation layer로 볼 수 있다. `#ROM` `#optimization` `#surrogate`
- [Control-oriented cluster-based reduced-order modelling](https://arxiv.org/abs/2604.25474) — cluster network model을 control parameter 함수로 일반화해 unseen operating condition의 reduced dynamics를 합성. flow control surrogate로 실험해볼 만하다. `#control` `#ROM` `#dynamical-systems`

### CAD/CAE agents & MCP workflows

- [sim-benchmark](https://github.com/svd-ai-lab/sim-benchmark) — OpenFOAM/LTspice task를 실제 solver artifact와 source provenance로 검증하는 industrial simulation agent benchmark. LLM-as-judge가 아니라 produced files와 extraction command를 다시 실행해 KPI를 채점한다는 점이 중요하다. `#OpenFOAM` `#agents` `#benchmark`
- [Tessa Labs freecad-mcp](https://github.com/tessalabs-space/freecad-mcp) — FreeCAD를 MCP로 노출해 parametric CAD, design sweep, drawing, defeaturing, mesh/export, BC tagging, CAE handoff를 agent-callable surface로 만드는 프로젝트. CAD-to-CAE 자동화 interface 설계 참고용으로 좋다. `#FreeCAD` `#MCP` `#CAE`
- [FoamPilot](https://github.com/elotech47/foampilot_csc7644) — natural-language OpenFOAM agent prototype. official tutorial case를 검색·복사·수정하는 방식이라, “처음부터 파일 생성”보다 안전한 OpenFOAM automation pattern을 보여준다. `#OpenFOAM` `#agentic-workflow` `#CFD`

## Best Find of the Day

**sim-benchmark**가 오늘의 best pick. CFD agent가 멋진 설명을 하는 것보다, 실제 OpenFOAM case를 만들고 solver를 돌린 뒤 KPI가 어떤 artifact에서 어떻게 추출됐는지 재현 가능해야 한다. VA의 intelligent engineering workflow 관점에서는 “agent benchmark = reasoning benchmark”가 아니라 “solver-grounded artifact benchmark”여야 한다는 기준을 잘 잡아준다.

## Possible Research Ideas

1. **OpenFOAM agent grading harness:** 작은 canonical cases(lid-driven cavity, heated cavity, pipe flow)를 대상으로 result.json + source provenance schema를 만들고, agent가 낸 KPI를 재실행 검증한다.
2. **Sensor-to-ROM control demo:** rt-EBIV 논문 구조처럼 coarse field/sparse probes → POD coordinates → linear/Kalman temporal regularization → simple control objective까지 연결한 toy workflow를 만든다.
3. **FreeCAD MCP CAE handoff checklist:** generated CAD에 대해 defeaturing, named face/BC tagging, STEP/STL/UNV export, meshability check를 자동 검증하는 최소 checklist를 설계한다.
